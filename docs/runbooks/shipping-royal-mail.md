# Royal Mail — отправка UK-заказа (полу-ручной флоу)

> **Когда применять:** заказ с UK-адресом. Доставка для покупателя **бесплатная** (free UK shipping).
>
> **Состояние интеграции:** Click & Drop на **Basic-тарифе** даёт API только для **создания заказа**, но **не для скачивания этикетки** (PDF доступен только в web-интерфейсе C&D). Поэтому после оплаты заказа система автоматически создаёт черновик в C&D, а оператор затем **вручную печатает этикетку** через сайт `www.royalmail.com/business/shipping/manage-shipping/click-and-drop` и заливает её обратно в админку YNOT.
>
> **Когда поменяется:** при апгрейде Click & Drop на тариф **Advanced** (~£20/мес) появится `Label API` → флоу станет полностью автоматическим как у DHL. Сейчас это **TODO** — пока работаем вручную.
>
> **Сколько занимает:** ~5–7 минут на заказ.

---

## Чек-лист "перед стартом"

Один раз перед записью обучающего видео убедись:

- [ ] У тебя установлен **термопринтер для этикеток A6** (или обычный A4 принтер — этикетка займёт верхнюю четверть листа)
- [ ] В Click & Drop оплачен баланс (с пустым балансом этикетку не сгенеришь)
- [ ] Адрес отправителя в Click & Drop = `[твой склад/адрес отгрузки]`, postcode **SW7 5QG**
- [ ] У тебя есть **бумажная коробка/конверт** правильного размера

---

## Как работает связка YNOT ↔ Click & Drop

1. **Покупатель оплатил** заказ на сайте YNOT
2. Background worker автоматически:
   - Создаёт `Shipment` в БД (carrier=`ROYAL_MAIL`)
   - Зовёт Click & Drop API → создаёт **черновик заказа в C&D dashboard**
   - В YNOT в поле `trackingNumber` записывается **RM Order ID** (например `00012345`) — это **ID черновика в C&D**, не финальный tracking number
   - Поле `labelGeneratedAt` остаётся **пустым** → в админке покажется как "label not generated yet"
3. Оператор открывает заказ в админке YNOT и видит:
   - Shipment: `ROYAL_MAIL · 00012345` (это и есть RM Order ID, по нему найдёшь черновик в C&D)
   - **Manual label upload** форма — будет видна, потому что labelGeneratedAt пустой
4. Оператор печатает этикетку из C&D, получает реальный tracking number, загружает PDF в админку

## Сценарий

### Шаг 1. Открой PAID заказ в админке
1. `https://staging.ynotlondon.com/admin/orders`
2. Найди заказ со статусом `PAID` и **Ship country = GB**
3. Кликни на номер заказа

### Шаг 2. Запомни RM Order ID
В секции **Shipments**:
```
ROYAL_MAIL · 00012345 ← это RM Order ID (5-значный)
Label: not generated yet
```
Скопируй RM Order ID (`00012345`) — он понадобится в Click & Drop.

### Шаг 3. Открой Click & Drop в новой вкладке
1. https://www.royalmail.com/business/shipping/manage-shipping/click-and-drop
2. Login (учётка YNOT, баланс должен быть положительный)
3. **Dashboard → Orders** (или **Orders to process**)
4. В поиске или фильтре введи **RM Order ID** (`00012345`) → откроется черновик заказа

> Если черновик **не появился** — значит API не смог создать заказ (баланс пустой, неверный адрес, или сбой). Создавай вручную в C&D — кнопка **Add an order → Manual order**.

### Шаг 4. Сверь данные в C&D
В черновике проверь что подтянулось из YNOT:
- Имя получателя
- Адрес доставки
- Postcode
- Описание содержимого (название товара)
- Вес (важно для тарификации!)

**Если веса нет** — введи вручную. Кожаная куртка ≈ 1.2–1.8 кг (зависит от модели; для точности взвесь на кухонных весах).

### Шаг 5. Выбери сервис и распечатай этикетку
1. **Postage service:** `Royal Mail Tracked 48` (или `Tracked 24` если хочешь быстрее — стоит дороже)
2. **Apply** → C&D рассчитает стоимость и спишет с баланса
3. Жми **Print → Label** (PDF откроется)
4. **Сохрани PDF локально** (Save as → `YN-2026-00021-label.pdf` или похоже)
5. На этой же странице C&D появится **финальный tracking number** (длинный, начинается с `AB123456789GB`) — **скопируй его**

### Шаг 6. Загрузи этикетку в админку YNOT
1. Возвращайся во вкладку YNOT админки на странице заказа
2. Прокрути вниз до блока **Manual label upload**
3. Заполни:
   - **Shipment**: выбери из дропдауна (`ROYAL_MAIL · 00012345`)
   - **Tracking number**: вставь **финальный RM tracking** из C&D (`AB123456789GB`, **не RM Order ID**)
   - **Label PDF**: выбери сохранённый файл
4. Жми **Save manual label**
5. Страница обновится: теперь `Label: 2026-05-14 10:23` и появится кнопка **Print & despatch**

### Шаг 7. Печать и упаковка
1. **Print & despatch** → откроется страница с **встроенным PDF этикетки** (тот что ты только что залила)
2. `⌘+P` → принтер → A4 или термоэтикетка → **Print**
3. Положи товар в коробку с упаковкой YNOT (tissue, бирка, благодарственная карточка)
4. Наклей этикетку на самую большую плоскую грань коробки. Штрихкод должен быть **полностью виден, без заломов**

### Шаг 8. Отметь как отгруженное
Только **после** реальной передачи коробки:

1. На странице `/admin/orders/<id>/ship` найди блок **Mark as despatched**
2. Галочка напротив shipment'а → **Mark as despatched**
3. Что произойдёт автоматически:
   - Статус заказа → **IN_TRANSIT**
   - Покупателю улетит email "Your order has shipped" с RM tracking number
   - В Click & Drop черновик автоматически отмечен как Despatched когда курьер отсканирует штрихкод

### Шаг 9. Передача курьеру
- **Post Office** — отнеси коробку. Возьми **proof of postage** (бумажный чек) — пригодится если посылка потеряется
- **Click & Drop pickup** — если включила pickup-расписание в C&D, курьер заедет сам

## Частые ошибки

| Симптом | Причина | Решение |
|---------|---------|---------|
| В C&D **нет черновика** с RM Order ID из YNOT | API не дозвонился (баланс пуст / адрес кривой) | Создай заказ вручную: C&D → **Add an order → Manual order** |
| Tracking number в C&D **не показывается** | Не выбрала Postage service / не нажала Apply | Вернись на черновик, выбери Tracked 48, Apply, потом Print |
| После Save manual label кнопка Print & despatch неактивна | PDF не загрузился (network error) | Перезагрузи страницу, проверь что `Label: <date>` появился. Если нет — повторно загрузи |
| Покупатель пишет "tracking не работает" | RM обновляет трекинг с задержкой 4–24ч после приёма курьером | Не паниковать. Если через 48ч всё ещё нет апдейта — звони в RM Customer Service |
| Залила не тот PDF | Перепутала файлы | Просто загрузи правильный через Manual label upload ещё раз — перезапишет |

## Что упрощается при апгрейде на Click & Drop Advanced

При переходе на **Advanced tier** (~£20/мес) Royal Mail откроет **Label API endpoint**. Тогда:
- Worker сам скачает PDF этикетки сразу после createShipment
- `labelGeneratedAt` заполнится автоматически
- Manual label upload **больше не нужен**
- Флоу станет таким же как DHL: PAID → Print & despatch → Mark as despatched (без вкладки C&D)

Когда решишь апгрейдить — дай знать, я добавлю env-флаг `ROYAL_MAIL_TIER=advanced` и обновлю код чтобы он перестал бросать `RoyalMailLabelApiUnavailableError`. Документации на 1 PR.

## Видео-демо: что обязательно показать

Для команды (5–7 минут ролика):

1. (30 сек) Какой заказ берём — найти PAID + GB на `/admin/orders`
2. (30 сек) Где **RM Order ID** в админке YNOT
3. (2 мин) В Click & Drop: найти черновик по ID → проверить адрес и вес → выбрать сервис → Print → сохранить PDF + скопировать tracking
4. (1 мин) Manual label upload в админке YNOT: shipment, tracking, PDF, Save
5. (1 мин) Print & despatch → реально печатаем → клеим на коробку
6. (30 сек) Mark as despatched → показать email который пришёл клиенту
